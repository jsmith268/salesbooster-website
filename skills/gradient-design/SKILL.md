Design gradients that read as deliberate and brand-specific — not as Midjourney-default purple-to-cyan. This skill produces ready-to-paste CSS using modern CSS Color 4 features (`oklch`, `color-mix`, `in oklch` interpolation).

## MANDATORY PREPARATION

Invoke `/impeccable` and follow its **Context Gathering Protocol**. Additionally gather: brand color tokens, surface (hero / button / card / divider), light/dark mode, target mood (warm/cool/neutral/dramatic), and whether the gradient will sit behind crisp content (text, UI) or atmospheric content (illustration, photography).

For an entire visual system, partner with `/colorize`. For graphic compositions on top of the gradient, partner with `/graphics-forge`.

## Why most gradients look AI-generated

The "AI gradient" tell comes from a fixed set of mistakes. Recognize and avoid them:

1. **Linear two-stop in sRGB.** The default `linear-gradient(#A, #B)` interpolates through muddy mid-tones (the "gray dead zone" between complementary hues).
2. **Naive complementary pairings.** Purple-to-cyan, pink-to-blue — exhausted tropes.
3. **No off-axis anchor.** Premium gradients have a warm or saturated point pulling the eye somewhere unexpected.
4. **No texture.** Smooth gradients band on real screens; grain/noise sells them as physical.
5. **One layer, one direction.** Real designed gradients are 2–4 layered gradients, often combining linear + radial + conic.

Decision check: if your gradient could be regenerated by typing "purple to blue gradient" into Midjourney, redesign it.

## OKLCH and perceptual interpolation

CSS Color 4 has universal browser support as of 2025+. Use it.

```css
/* Perceptually uniform — no muddy midpoint */
background: linear-gradient(in oklch, #ff5e8a, #5b3fff);

/* Force long way around the color wheel — rich rainbow sweep */
background: linear-gradient(in oklch longer hue, #ff5e8a, #00d4ff);

/* Default direction, clean two-color blend */
background: linear-gradient(in hsl shorter hue, #ff0, #f0f);
```

**Rule:** default to `in oklch` for any gradient between two saturated colors. Use sRGB only when blending from/to white, black, or transparent.

## Multi-stop signature gradients

3–5 stops, with an **off-axis warm anchor**. The warm anchor is what separates a designed gradient from a generated one.

```css
.signature-twilight {
  background: linear-gradient(135deg in oklch,
    #1a0b2e   0%,   /* deep violet base */
    #3d1e6d  28%,   /* mid violet */
    #c54bdb  55%,   /* magenta heart */
    #ff6b9d  78%,   /* pink lift */
    #ffb37c 100%);  /* warm anchor — the "moment" */
}
```

The amber endpoint is the move. Generic AI gradients stop at the magenta.

## CSS mesh-gradient approximation (Stripe formula)

True mesh gradients aren't in CSS yet. Approximate by stacking radial gradients on a deep base:

```css
.mesh {
  background-color: #0b0a1a;
  background-image:
    radial-gradient(at 20% 20%, oklch(0.70 0.25 350) 0px, transparent 50%),
    radial-gradient(at 80% 10%, oklch(0.65 0.22 280) 0px, transparent 50%),
    radial-gradient(at 70% 80%, oklch(0.75 0.20  30) 0px, transparent 50%),
    radial-gradient(at 10% 90%, oklch(0.60 0.18 200) 0px, transparent 50%);
}
```

Four off-axis radial blobs over a deep base. This is the Stripe-homepage formula.

## Conic gradients (rotational light)

```css
.conic-halo {
  background: conic-gradient(from 220deg at 65% 35% in oklch,
    oklch(0.70 0.25 350),
    oklch(0.65 0.22 280),
    oklch(0.75 0.20  30),
    oklch(0.70 0.25 350));
  filter: blur(80px);
}
```

**Always blur conic gradients heavily for ambient lighting** — sharp conics look like color wheels.

## Noise overlay (kills banding, sells the gradient as physical)

```css
.gradient-with-grain {
  position: relative;
  background: linear-gradient(in oklch, #1a0b2e, #ff6b9d);
}
.gradient-with-grain::after {
  content: "";
  position: absolute; inset: 0;
  background-image: url("data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='200' height='200'><filter id='n'><feTurbulence baseFrequency='0.9' numOctaves='2'/></filter><rect width='100%' height='100%' filter='url(%23n)' opacity='0.5'/></svg>");
  mix-blend-mode: overlay;
  opacity: 0.18;
  pointer-events: none;
}
```

For hero-scale gradients, **bake** the noise to a tiled PNG (256×256) — running SVG turbulence full-bleed every frame is a battery killer.

## When NOT to use gradients

- Body text backgrounds (kills legibility).
- Buttons in dense UI (creates visual noise; flat or single-direction subtle gradient only).
- Anywhere that needs to feel utilitarian/data-dense (admin tools, dashboards).
- As a crutch for weak typography or layout.

## 2026-current CSS techniques

- **`color-mix()`** for programmatic palettes:
  ```css
  --tint: color-mix(in oklch, var(--brand) 60%, white);
  ```
- **Relative color syntax** for derived shades:
  ```css
  --brand-bright: oklch(from var(--brand) calc(l + 0.1) c h);
  ```
- **Wide-gamut color** (`display-p3`, `rec2020`) on OLED phones for richer saturation than sRGB allows:
  ```css
  background: linear-gradient(in oklch, color(display-p3 1 0 0.4), oklch(0.7 0.25 280));
  ```
- **Animated gradients** via `@property` — register custom properties as `<color>` to interpolate:
  ```css
  @property --g1 { syntax: "<color>"; inherits: false; initial-value: #5b3fff; }
  .anim { background: linear-gradient(135deg in oklch, var(--g1), #ff6b9d); transition: --g1 600ms; }
  .anim:hover { --g1: #c54bdb; }
  ```

## Named recipe library — paste-ready

```css
:root {
  /* Twilight — signature dark hero, violet to warm anchor */
  --grad-twilight: linear-gradient(135deg in oklch,
    #0b0a1a 0%, #2d1659 30%, #c54bdb 65%, #ffb37c 100%);

  /* Cinder — editorial dark, ember tones */
  --grad-cinder: linear-gradient(160deg in oklch,
    #1a0808 0%, #3d0f1e 35%, #c2410c 75%, #fbbf24 100%);

  /* Ocean Glass — cool, calm, B2B trust */
  --grad-ocean-glass: linear-gradient(180deg in oklch,
    #f0f7ff 0%, #b8d4f0 40%, #6ba3d0 80%, #2c5282 100%);

  /* Aurora — mesh, bright, energetic */
  --grad-aurora:
    radial-gradient(at 15% 20%, oklch(0.75 0.25 320) 0px, transparent 45%),
    radial-gradient(at 85% 15%, oklch(0.78 0.20 220) 0px, transparent 45%),
    radial-gradient(at 70% 85%, oklch(0.80 0.22  80) 0px, transparent 50%),
    radial-gradient(at 10% 80%, oklch(0.70 0.20 160) 0px, transparent 45%),
    #fafaff;

  /* Graphite — Vercel-style monochrome with warmth */
  --grad-graphite: linear-gradient(180deg in oklch,
    #0a0a0a 0%, #1f1f1f 40%, #2d2a26 80%, #3d3530 100%);

  /* Ember Halo — conic, behind blurred foreground */
  --grad-ember-halo: conic-gradient(from 200deg at 70% 30% in oklch,
    #ff6b35, #c2410c, #7c2d12, #c2410c, #ff6b35);

  /* Mint Drift — fresh, optimistic, not minty cliché */
  --grad-mint-drift: linear-gradient(115deg in oklch,
    #f0fdf4 0%, #86efac 40%, #14b8a6 75%, #0e7490 100%);
}
```

## Decision rules

- **Two saturated colors?** Always `in oklch`.
- **Hero background?** Always multi-stop or layered, always with grain.
- **Button or card?** Subtle (8–15% lightness range), single direction, no warm anchor.
- **Dark mode?** Anchor with at least one warm hue to avoid "cold tech" feeling.
- **Light mode?** Anchor with at least one saturated cool hue to avoid washing out.
- **Brand has one core hue?** Build the gradient as variations of that hue (`oklch(from var(--brand) ...)`) plus a single contrasting anchor — not a rainbow.

## Reviewing existing gradients

When asked to review, check:

1. Is it two-stop sRGB? → upgrade to multi-stop `in oklch`.
2. Does it have a warm/saturated anchor? If not, add one off-axis.
3. Is there grain/noise? If hero-scale and missing, add.
4. Does it band visibly on dark surfaces? Diagnose: missing noise, sRGB interpolation, or under-saturated stops.
5. Does it conflict with brand color? Either pull the brand into one of the stops or simplify the gradient.

## Output format — what to deliver

1. **The CSS** — paste-ready, scoped to the user's project (CSS variables, Tailwind v4 layer if applicable).
2. **Why this gradient** — short justification: which mood, which anchor, why these stops.
3. **Surface notes** — where it goes, what should sit on top, contrast considerations.
4. **Performance** — when to bake to image, when to keep CSS-native.
5. **Pairing recommendation** — does it want grain (`/image-treatment` recipe), graphics on top (`/graphics-forge`), or stand alone?

## Reference brands

- **Stripe** — mesh gradients with 4-color anchors, always with a warm tone present.
- **Linear** — dark dual-anchor gradients (deep violet + teal), heavy grain.
- **Vercel** — largely monochrome gradients, conic light highlights.
- **Apple** — subtle, often display-P3, mostly behind product photography.
- **Figma** — bright multi-stop gradients with a clear "signature" hue at center.
- **Arc** — playful candy gradients with strong identity.

The tell of a great gradient: a *moment* — a specific point of warmth or saturation that feels chosen, not interpolated.
